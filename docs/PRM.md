# Chrome拡張機能 開発指示書
## 「Amazon × NotebookLM ソース自動追加ツール」

---

## 📌 プロジェクト概要

AmazonのKindle・書籍ページを開いたとき、その書籍タイトルをもとに関連する要約記事・レビュー記事・YouTube解説動画を自動収集し、NotebookLMのソースとして一括追加するChrome拡張機能を開発してください。

---

## 🎯 ユーザーストーリー

> 「Amazonで気になる本を見つけたとき、拡張機能のボタンを1回押すだけで、その本に関するレビュー・要約・YouTube解説動画がNotebookLMに自動追加され、本を読む前に内容の全体像を掴めるようにしたい」

---

## ✅ 機能要件

### 1. Amazonページの検出・タイトル取得
- Amazonの書籍・Kindle商品ページを開いているときだけ、拡張機能アイコンをアクティブ（カラー表示）にする
- ページのDOMから書籍タイトルを自動取得する
- 取得できない場合はユーザーに手入力を促す

### 2. 関連コンテンツの検索・収集
以下の2種類のソースを収集する：

**Webページ（記事）**
- 「{書籍タイトル} 要約」
- 「{書籍タイトル} レビュー」
- 「{書籍タイトル} まとめ」
- 上位5〜10件のURLを収集

**YouTube動画**
- 「{書籍タイトル} 要約」
- 「{書籍タイトル} 解説」
- 上位3〜5件の動画URLを収集

### 3. ソース選択UI（ポップアップ）
収集した候補を一覧表示するポップアップUIを表示する：
- 各ソースにチェックボックスを設け、デフォルトは全選択
- Web記事とYouTube動画をセクション分けして表示
- タイトル・URLのプレビューを表示
- 「全選択」「全解除」ボタンを設置
- 「自動追加モード（全件追加）」と「選択追加モード」を切り替えられるトグルを設置

### 4. URLリストのクリップボードコピー
- ポップアップに「URLをコピー」ボタンを設置する
- チェックボックスで選択中のURLのみをコピー対象とする
- コピー形式は1行1URLのプレーンテキスト（NotebookLMへの手動貼り付けを想定）
- コピー完了時にボタンのラベルを一時的に「✅ コピーしました」に変えてフィードバックを表示する
- **用途：** NotebookLMのUI変更などで自動追加が機能しなくなった場合のフォールバックとして使用する

### 5. NotebookLMへの自動追加

選択されたURLをNotebookLMに自動追加する。実装は以下の**2段階戦略（Tier構成）**で行うこと。

#### Tier 1：フェッチインターセプト（優先）
- NotebookLMがブラウザ内部で送るAPIリクエストをcontent scriptでインターセプトし、同じリクエストを追加URLで再現する
- DOMを操作しないため高速・安定しており、UIの変更に影響されない
- まずこの方法を試み、成功した場合はTier 2を使わない

#### Tier 2：DOMオートメーション（フォールバック）
- Tier 1が利用できない場合に自動的に切り替わる
- 具体的なフロー：
  1. `https://notebooklm.google.com` を新しいタブで開く（すでに開いている場合はそのタブを使用）
  2. 対象のノートブックを選択または新規作成する画面に遷移する
  3. 「ソースを追加」ボタンを操作し、URLを順次入力・追加する
- セレクタはCSS・テキスト・ARIAラベル・ロールの複数戦略で解決し、UIが多少変更されても動作するよう実装すること
- DOM操作部分はモジュール化し、コメントを充実させること（仕様変更時の修正箇所が一目でわかるように）

#### 共通仕様
- 追加中は進捗バー（例：「3 / 8 件追加中...」）をポップアップに表示する
- 全件追加完了後、ポップアップに「✅ {N}件のソースを追加しました」と表示する
- エラーが発生した場合は該当URLをスキップしてログに記録し、最後にまとめてユーザーに通知する
- 失敗したURLはポップアップ上で再試行できるボタンを設置する

## 🔧 技術要件

### 対応ブラウザ
- Google Chrome（Manifest V3準拠）
- ※ 将来的にEdge対応も視野に入れ、ブラウザ固有APIへの依存を最小化すること

### 検索APIについて
- Google Custom Search API または SerpAPI を使用してWeb記事を収集する
- YouTube Data API v3 を使用して動画を収集する
- APIキーはユーザーが拡張機能の設定画面から入力・保存できるようにする（chrome.storage.syncを使用）
- APIキーが未設定の場合は、設定画面への誘導メッセージを表示する

### NotebookLM操作について
- NotebookLMに公式APIは存在しないため、content scriptによるDOMオートメーションで実装する
- セレクタが変更された場合に備え、DOM操作部分はできるだけモジュール化・コメントを充実させること

### ファイル構成（推奨）
```
extension/
├── manifest.json
├── popup/
│   ├── popup.html
│   ├── popup.js
│   └── popup.css
├── content_scripts/
│   ├── amazon.js                    # Amazonページからタイトル取得
│   ├── notebooklm.js               # NotebookLMのDOM操作（Tier 2）
│   └── notebooklm-interceptor.js   # フェッチインターセプト（Tier 1）
├── background/
│   └── service_worker.js
├── options/
│   ├── options.html     # APIキー設定画面
│   └── options.js
└── icons/
    ├── icon16.png
    ├── icon48.png
    └── icon128.png
```

## 🎨 UIガイドライン

- シンプルで直感的なデザイン
- 横幅380px程度のポップアップ
- ローディング中はスピナーを表示
- 日本語UIを基本とする
- 配色はNotebookLMのブランドカラー（青・白系）に合わせる

---

## ⚠️ 考慮事項・制約

- NotebookLMのDOM構造は予告なく変更される可能性があるため、オートメーション部分は壊れやすいことをコード内にコメントで明記すること
- Amazon.co.jp と Amazon.com の両方のURLパターンに対応すること
- 検索結果に低品質なページが混入しないよう、ドメインフィルタリング（スパムサイト除外）の仕組みを設けること
- ユーザーのGoogleアカウントへのログイン状態はNotebookLM側で管理されるため、拡張機能側では認証処理を行わないこと

## 📦 最初のフェーズで作るもの（MVP）

まず以下の最小構成で動作するバージョンを作成してください：

1. Amazonページでのタイトル取得
2. YouTube検索による動画URL収集（3件）
3. Web記事検索による記事URL収集（5件）
4. ポップアップでの候補一覧表示とチェックボックス選択
5. URLリストのクリップボードコピー機能
6. NotebookLMを開いてURLを自動追加

設定画面・エラーハンドリングの充実・デザインの洗練は、MVPが動作確認できてから次のフェーズで行う。
